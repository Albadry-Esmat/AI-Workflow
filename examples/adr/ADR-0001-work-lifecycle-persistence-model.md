# ADR-0001 — Work Lifecycle Persistence Model

**Status:** Accepted  
**Date:** 2026-06-20  
**Deciders:** Pipeline orchestrator  

---

## Context

The work-item system needed a persistence strategy for tasks, features, and bug reports
generated by the pipeline. Options considered:

1. In-memory only (lost on session end)
2. Database (PostgreSQL / SQLite)
3. Filesystem (Markdown files in `work-items/`)

## Decision

Store all work items as Markdown files in `work-items/` using a structured frontmatter
schema (YAML). No external database dependency.

## Consequences

**Positive:**
- Zero infrastructure — works in any environment
- Git-trackable: history, diffs, and blame are native
- Human-readable without tooling

**Negative:**
- No query/filter capabilities beyond grep/find
- Concurrent writes require coordination
- Scale is bounded by filesystem performance
