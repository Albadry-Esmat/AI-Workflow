# Architecture Decision Records

This directory holds ADRs generated by the `adr-generator` skill during pipeline runs.

ADRs are created automatically when the pipeline makes significant architectural choices,
tech stack selections, or design tradeoffs.

## Format

Each ADR file follows this naming convention:

```
ADR-NNNN-short-title.md
```

And uses the standard ADR template with sections:
- **Context** — the situation and forces at play
- **Decision** — what was decided
- **Consequences** — trade-offs and outcomes

## Example ADRs

See [`examples/adr/`](../../examples/adr/) for sample ADRs generated by the framework.

## Creating ADRs

Trigger the `adr-generator` skill by asking the pipeline to document a decision:

```
"write an ADR for this decision"
"document why we chose <technology>"
"create an architecture decision record for <tradeoff>"
```
